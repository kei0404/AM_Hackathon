# CLAUDE.md 情報収集インタビュー

あなたはプロジェクトマネージャーとして、プロジェクトの基盤となる `CLAUDE.md` を作成するために必要な情報をユーザーから聞き取ります。
spec.md を参照して

## 実行ステップ

1. **プロジェクトの現状把握**
   - 現在のディレクトリ構造や主要なファイル（`README.md`, `package.json`, `pyproject.toml`, `requirements.txt` 等）を確認し、自動的に判断できる情報（プロジェクト名、技術スタックなど）をまとめます。

2. **不足情報の特定**
   - `generate-claude.md` で定義されている以下の項目について、不足している情報を特定します。
     - メタデータ（プロジェクト名、バージョン、ライセンス）
     - プロジェクトの目的・背景 (Purpose/Context)
     - 主要な機能仕様 (Core Functionality Specifications)
     - 開発における前提条件や独自のルール (Top-Level Rules)
     - AI（Claude Code）への指示指針 (Prompt Design Guidelines)
     - 注意点、制約、禁止事項 (Notification/Constraints/DONT'S)

3. **インタビューの実施**
   - ユーザーに対し、不足している情報を質問します。
   - 一度に全ての質問を投げず、関連する項目ごとに（例：まずは目的と機能について、次にルールについて等）、答えやすい粒度で質問してください。
   - 質問は一問一答形式で実施してください。
   - すでにファイルから読み取れた情報がある場合は、「〜と理解しましたが、相違ないですか？」と確認する形式にしてください。

4. **情報の整理と次への提案**
   - 全ての情報を収集し終えたら、内容を要約して提示します。
   - 収集した情報は、`project-info-*.md`として、./docs/project_info に保存してください。
   - 最後に、収集した情報をもとに `.claude/commands/create-claude.md` を実行して `CLAUDE.md` を生成することを提案してください。

5. **禁止事項に関する質問**
   - プロジェクトにおける禁止事項について詳細に確認する
   - リポジトリの管理について
   - 未承認のプロンプトの取り扱い
   - 秘密情報について
   - 環境変数などの取扱いについて
   - ハードコーディングについて
   - テストや検証について

## 注意事項
- ユーザーの手間を最小限にするため、コードから推測できることは推測し、確認のみを求めるようにしてください。
- 専門用語の定義や、プロジェクト特有の「やってはいけないこと」などを重点的に聞き出すと、より質の高い `CLAUDE.md` になります。
- 丁寧かつ簡潔な日本語で対話してください。
