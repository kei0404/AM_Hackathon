# LLM の想定レスポンス

次に会話の流れを変更する。

# 前提
これはナビゲーションアプリのバックエンドアプリです。

# 会話の挙動
1. ユーザーから現在地情報を受け取ったら「どこに行きたいですか？」と質問して、目的地情報を取得する。
2. 目的地の情報を取得したら、次に「他に行きたいところ、やってみたいことはありますか？」と質問を返す。
3. その後ユーザーから得た情報に対して、RAGの検索をかけ、検索結果を取得する。
4. と検索結果と会話のキャッシュをコンテキストにして、ユーザーに目的地以外の新しい訪問先を提案するように動作するようにして下さい。

# 更なる変更点
- ユーザーからの"行きたいところ・やってみたいこと"に関する入力に対しての提案は３つまでにする
- LMには、１つ目の提案に対して「ここに行きますか？」と応答させ、ユーザーが賛成すれば、そこを目的地とする。
- ユーザーが１つ目で賛成しなければ２つ目の提案を返す。２つ目の提案でも賛成しなければ３つ目を返す。
- ３つの提案全てで賛成しなければ、「他に希望はありますか？」とLMに質問させる。
- 他の希望があれば、もう1度「行きたいことろ・やりたいことはありますか？」と質問する。希望がなければ、目的地に直行する。
- 旅程が決定したら、「目的地は〇〇、立ち寄る場所は〇〇です」と応答をLMからユーザーに返す。
